# 결정 사항

구현 전 선택이 필요한 항목들입니다.

---

## 1. AI 프로바이더

### 왜 필요한가?

데스크톱 펫의 핵심 기능은 **질문에 답변하는 것**입니다.

```
사용자: "오늘 날씨 어때?"
펫: "서울은 현재 맑고 기온은 5도입니다."

사용자: "이 코드 뭐가 잘못됐어?"
펫: "3번째 줄에 세미콜론이 빠졌네요."
```

이 답변을 생성하려면 AI 모델이 필요합니다. 어떤 AI를 사용할지에 따라:

- **비용**: 무료 vs 월 $20+
- **속도**: 로컬(즉시) vs API(네트워크 지연)
- **품질**: 단순 답변 vs 복잡한 추론
- **프라이버시**: 내 질문이 외부 서버로 전송되는가?

### 옵션

| 옵션                      | 장점                            | 단점                             |
| ------------------------- | ------------------------------- | -------------------------------- |
| **A) OpenAI (GPT)**       | 가장 보편적, 문서 풍부          | 유료, API 키 필요                |
| **B) Anthropic (Claude)** | 긴 컨텍스트, 안전성             | 유료, API 키 필요                |
| **C) Ollama (로컬)**      | 무료, 오프라인 가능, 프라이버시 | 설치 필요, 성능은 PC 사양에 의존 |
| **D) 다중 지원**          | 유연함, 사용자 선택 가능        | 구현 복잡도 증가                 |

## 2. 전역 단축키

### 왜 필요한가?

현재 펫은 **투명 영역 클릭이 통과**됩니다. 이게 장점이지만, 문제가 있습니다:

```
[문제 상황]
1. 펫이 화면 중앙에 있음
2. 펫 뒤에 있는 버튼을 클릭하고 싶음
3. 펫의 불투명한 부분이 버튼을 가리고 있음
4. 클릭이 펫에게 가서 버튼을 누를 수 없음!
```

**해결책**: 단축키로 모드 전환

- `Passthrough 모드`: 현재처럼 투명 영역만 통과
- `Interactive 모드`: 펫 전체가 클릭 가능 (데스크톱 접근용)

단축키를 누르면 펫이 잠시 "유령 모드"가 되어 뒤에 있는 것을 클릭할 수 있습니다.

### 옵션

| 옵션                    | 설명                              |
| ----------------------- | --------------------------------- |
| **A) Cmd+Shift+D**      | D = Desktop pet                   |
| **B) Cmd+Shift+P**      | P = Pet                           |
| **C) Cmd+Option+Space** | 직관적, 스포트라이트 느낌         |
| **D) 기타**             | 원하는 조합 직접 입력: **\_\_\_** |

## 3. 인터랙티브 모드 시각적 표시

### 왜 필요한가?

모드가 전환되면 사용자가 **현재 어떤 모드인지 알아야** 합니다.

```
[시각적 표시가 없으면]
- 단축키 눌렀는데... 바뀐 건가?
- 지금 클릭하면 펫한테 가나? 뒤로 가나?
- 혼란 → 잘못된 클릭 → 짜증
```

```
[시각적 표시가 있으면]
- 단축키 누름 → 펫이 반짝거림 → "아, 인터랙티브 모드구나"
- 확신을 갖고 클릭
```

### 옵션

| 옵션               | 설명                     | 예시                       |
| ------------------ | ------------------------ | -------------------------- |
| **A) 글로우 효과** | 캐릭터 주변에 은은한 빛  | 네온사인처럼 빛남          |
| **B) 테두리**      | 캐릭터 주변에 컬러 보더  | 선택된 아이콘처럼          |
| **C) 투명도 변화** | 모드 전환 시 살짝 밝아짐 | 활성화된 느낌              |
| **D) 복합**        | 글로우 + 투명도 조합     | 확실한 피드백              |
| **E) 없음**        | 커서 변화만으로 표시     | 미니멀, 하지만 놓치기 쉬움 |

## 4. 채팅 UI 스타일

### 왜 필요한가?

펫에게 질문하려면 **입력창**이 필요하고, 답변을 보여줄 **출력 영역**이 필요합니다.

```
[채팅 흐름]
1. 사용자가 펫 클릭 (또는 단축키)
2. 입력창이 나타남
3. "오늘 할 일 뭐야?" 입력
4. 펫이 답변을 보여줌
5. 입력창 닫힘 (또는 계속 대화)
```

이 UI가 어떻게 생겼느냐에 따라 **사용감**이 완전히 달라집니다.

### 옵션

| 옵션             | 설명                          | 적합한 경우             |
| ---------------- | ----------------------------- | ----------------------- |
| **A) 말풍선**    | 캐릭터 옆에 붙는 작은 말풍선  | 짧은 대화, 펫 느낌 살림 |
| **B) 하단 패널** | 캐릭터 아래에 펼쳐지는 입력창 | 긴 대화, 히스토리 필요  |
| **C) 별도 창**   | 독립된 채팅 윈도우            | 본격적인 AI 어시스턴트  |
| **D) 미니멀**    | 입력만 있고 응답은 말풍선으로 | 간단한 질의응답         |
